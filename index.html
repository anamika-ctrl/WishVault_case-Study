Core Problem Statement

Online shoppers in India have no unified way to track, organise, or share their wishlists across the multiple e-commerce platforms they actively use. This leads to missed price drops, duplicate purchases, and a broken gift-giving experience.

Four Pain Points Identified

1 Wishlist fragmentation

Saved items are siloed across 4–8 different apps. Users have no single view of what they want. A saree on Myntra, headphones on Amazon, and a laptop on Flipkart exist in three separate universes.

2 Missed price drops

Items go on sale during major events. Users have no way to know when a saved item drops in price unless they manually check every day. Most price-sensitive purchases are missed.

3 No cross-platform sharing

Sharing a wishlist requires the recipient to have the exact same shopping app. There is no standard way to share a mixed wishlist, resulting in duplicate gifts, wrong sizes, and returns.

4 High friction to save items

The current workaround is copying a product URL and pasting it somewhere else. There is no native share-to-wishlist flow on any major Indian platform, making a 1-second task take minutes.

Interview Insights

  • Users don't mind a separate app — they want it to be the last one they open before buying.
  • Price drop alerts are the single highest-value feature across all personas.
  • The Chrome extension save flow is the mental model users already have (like Pinterest).
  • Gift list sharing is uniquely tied to Indian festivals — not served well currently.
  • Affiliate monetisation is invisible to users and is not a reason to avoid the product.

Dropped

Concept A: Mobile App

Standalone mobile app where users paste product URLs to save items.

Verdict: URL pasting is 7+ steps across apps. 4 of 5 users abandoned after the first save attempt.

Dropped

Concept B: WhatsApp Bot

Share product link to a WhatsApp number. Bot saves and sends price alerts.

Verdict: WhatsApp API costs and Meta policy restrictions make this fragile at scale. Platform dependency.

Selected

Concept C: Chrome Ext.

Extension injects a floating Save button. 1-click capture. Price tracking runs server-side.

  • Lowest friction — no copy-paste
  • Works without API access
  • Client-side DOM reading is safer
  • Natural invisible monetisation

04 — Design

Building the product

Core System Architecture

Click Save → Content Script reads DOM → POST to API w/ OAuth → PostgreSQL Save + BullMQ Job → Re-check 6-24 hrs → Push Alert on Drop

Chrome Extension — Key Design Decisions

1. Manifest V3 & Remote Config

DOM selectors are stored in a remote config endpoint, not hardcoded. When Amazon updates HTML, we fix the selector server-side without a 3-day Chrome Web Store re-submission.

2. Injected Save Button

Instead of requiring users to click the extension icon, a floating "Save" button is injected directly onto product pages. Reduces save flow from 3 steps to 1 step. Doubled saves in testing.

3. Google OAuth via chrome.identity

Most secure approach, allowing reliable linking of saves to the user's account across mobile and web. Tokens validated server-side on every request.

4. Open Graph Fallback

For unknown sites, the extension falls back to Open Graph tags (og:title, og:price, og:image). Ensures WishVault works on hundreds of sites from day one.

Key Learnings

1. API approval is a hard product dependency

Amazon's PA API takes 2–4 weeks to be approved, gating all price features. The most important early action is submitting the application before writing code.

2. Selector config must live server-side

Hardcoding CSS selectors means a Chrome Store review cycle every time Amazon redesigns. A remote config makes fixes instantaneous.

3. Client-side reading is a moat

Reading what the user sees in their browser is legally and technically advantageous over server-side scraping (bypassing bot detection).

4. The "aha moment" is the first price alert

Users engage after saving, but are retained after receiving an alert. Onboarding must maximise the chance of a price drop arriving within 7 days.

5. Affiliate model aligns incentives

Commission is invisible to users and aligns with retailers. We can grow without friction or asking users for money until they see real value.
