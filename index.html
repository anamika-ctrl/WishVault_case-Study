01 — Problem

Shoppers are drowning
in disconnected wishlists

Indian online shoppers juggle 6–8 platforms. Wishlists live in silos, prices change without notice, and gift-giving is broken because lists aren't shareable across apps.

🗂️

Wishlist fragmentation

Saved items are spread across Amazon, Flipkart, Myntra, Nykaa — no single view of what you actually want.

📉

Missed price drops

Items go on sale during Big Billion Days or Republic Day Sales, but users don't find out until it's too late.

🔗

No cross-platform sharing

Sharing a wishlist requires the recipient to have the same app. Birthday and festival gift planning is unnecessarily hard.

⏱️

High friction to save items

Copy-pasting URLs between apps is clunky. There's no native "share to wishlist" flow on any Indian shopping platform.

02 — Solution

One click. Any site.
Saved forever.

WishVault is a Chrome extension + mobile app. The extension injects a Save button on every product page. One click captures the title, price, image and URL — synced instantly to the user's account.

1

User visits any product page

The extension is always active in the Chrome toolbar. A floating "Save to WishVault" button is injected onto every supported shopping site automatically — no icon-clicking required.

2

Extension reads the page DOM

A content script extracts product data using platform-specific selectors, with Open Graph tags as a universal fallback for any site we haven't explicitly mapped.

// Content script — Amazon example const title = document.querySelector('#productTitle')?.innerText const price = document.querySelector('.a-price-whole')?.innerText const image = document.querySelector('#landingImage')?.src const url = window.location.href
3

Authenticated save to backend

Data is sent to the API with the user's Google OAuth token. The backend deduplicates, saves to PostgreSQL, and enqueues a BullMQ job to track price changes every 6–24 hours.

// Single POST — saves item + starts price tracking POST /api/wishlist/save Authorization: Bearer <google_token> { title, price, image, url, platform }
4

Confirmation popup + wishlist view

The extension popup shows "Saved!" with the item thumbnail. Users can browse their full wishlist, see current prices across platforms, and get notified when something drops in price — all without leaving the current page.

05 — Key Learnings

What building this
actually taught me

Three decisions that shaped the product — each one a tradeoff between speed, reliability and legal risk.

01

API approval is a product dependency

Amazon's PA API takes 2–4 weeks to get approved. This single dependency gates Phase 2. Apply before writing a line of code — not after.

02

Selector configs must live server-side

E-commerce sites change their HTML constantly. Hardcoding selectors in the extension means a Chrome Web Store update (2–3 days) every time Amazon redesigns. Remote config = instant fixes.

03

Client-side reading is safer than server scraping

The extension reads what the user already sees in their own browser session. This is legally and technically much lower risk than server-side scraping — and a genuine product moat.
